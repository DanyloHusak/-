1)
Name, description: Бронювання квитка на виставку через онлайн-систему.
Actors: Користувач, Система.
Pre-condition: Користувач обрав тип квитка та ввів необхідні дані.
Post-condition: Користувач успішно забронював квиток, отримавши підтвердження.

1.Actor Action: Користувач вибирає опцію "Бронювання квитка" та обирає тип квитка і кількість.
System Response: Система перевіряє доступність обраних квитків і відображає їх доступність.

2.Actor Action: Користувач вводить персональні дані для бронювання квитка.
System Response: Система вимагає введення інформації про користувача (ім'я, прізвище, контактні дані).

3.Actor Action: Користувач підтверджує свої дані та бажану кількість квитків.
System Response: Система перевіряє дані користувача та відправляє запит на бронювання квитків у систему управління квитками.

4.Actor Action: Система підтверджує успішне бронювання квитків і відображає повідомлення про успішне завершення операції.
System Response: Система надсилає підтвердження бронювання користувачеві та відображає повідомлення про успішне бронювання.

Scenario: Користувач успішно бронює квиток на виставку через онлайн-систему.


2)
Name, description: Реєстрація нового користувача в системі.
Actors: Користувач, Система.
Pre-condition: Користувач знаходиться на сторінці реєстрації.
Post-condition: Новий користувач зареєстрований у системі.

1.Actor Action: Користувач вибирає опцію "Зареєструватися" на головній сторінці системи.
System Response: Система відображає форму для реєстрації нового користувача.

2.Actor Action: Користувач вводить свої особисті дані (ім'я, електронну пошту, пароль тощо) у відповідні поля.
System Response: Система перевіряє введені дані і вимагає введення всіх обов'язкових полів.

3.Actor Action: Користувач підтверджує свою реєстрацію, натискаючи кнопку "Зареєструватися".
System Response: Система зберігає дані нового користувача у базі даних і відображає повідомлення про успішну реєстрацію.

4.Actor Action: Користувач отримує лист на свою електронну пошту з посиланням для підтвердження реєстрації.
System Response: Система надсилає лист із запитом на підтвердження реєстрації користувачеві.

Scenario: Новий користувач успішно реєструється в системі.


3)
Name, description: Оформлення замовлення товару через онлайн-магазин.
Actors: Користувач, Система.
Pre-condition: Користувач має товари у кошику і введені контактні дані для доставки.
Post-condition: Замовлення товару успішно оформлене, користувач отримує підтвердження.

1.Actor Action: Користувач обирає товар, який бажає придбати, та додає його до кошика.
System Response: Система додає товар до кошика і відображає його деталі.

2.Actor Action: Користувач вибирає опцію "Оформити замовлення" і вводить свої контактні дані для доставки.
System Response: Система перевіряє введені дані і вимагає обов'язкове заповнення всіх полів форми.

3.Actor Action: Користувач підтверджує своє замовлення, натискаючи кнопку "Підтвердити замовлення".
System Response: Система зберігає дані замовлення у базі даних і відображає повідомлення про успішне оформлення замовлення.

4.Actor Action: Користувач отримує підтвердження замовлення на електронну пошту разом із деталями замовлення.
System Response: Система надсилає користувачеві лист із підтвердженням замовлення та деталями.

Scenario: Користувач успішно оформляє замовлення товару через онлайн-магазин.
