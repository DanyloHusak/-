Тест-кейси (Лаб 1)

User Story 1: Зміна пароля

Test Case: Успішна зміна пароля
Title: Успішна зміна пароля користувача
Description: Користувач може успішно змінити свій пароль, виконавши всі необхідні кроки і вимоги.
Preconditions: Користувач авторизований і знаходиться на сторінці налаштувань облікового запису.
Test Steps:
1) Введіть старий пароль.
2) Введіть новий пароль, який відповідає вимогам безпеки (наприклад, мінімальна довжина, різні типи символів).
3) Підтвердіть новий пароль.
4) Натисніть кнопку "Змінити пароль".
Expected Result: Система перевіряє коректність старого пароля, відповідність нового пароля вимогам безпеки, зберігає новий пароль і надсилає підтвердження через електронну пошту або SMS.


User Story 2: Відстеження тренувань

Test Case: Успішне створення тренувальної програми
Title: Успішне створення тренувальної програми
Description: Користувач може успішно створити тренувальну програму з правильним введенням даних.
Preconditions: Користувач авторизований і знаходиться у розділі створення тренувальної програми.
Test Steps:
1) Введіть вправи.
2) Вкажіть кількість повторень або навантаження для кожної вправи.
3) Вкажіть час виконання кожної вправи.
4) Натисніть кнопку "Зберегти програму".
Expected Result: Система зберігає тренувальну програму і вона стає доступною для подальшого використання.

Тест-кейси (Лаб 2)

Сценарій 1: Бронювання квитка на виставку через онлайн-систему

Test Case: Успішне бронювання квитка
Title: Успішне бронювання квитка
Description: Користувач може успішно забронювати квиток, виконавши всі необхідні кроки.
Preconditions: Користувач знаходиться на сторінці бронювання квитків у системі.
Test Steps:
1) Виберіть опцію "Бронювання квитка".
2) Оберіть тип квитка і кількість.
3) Введіть персональні дані для бронювання.
4) Виберіть спосіб онлайн оплати та введіть дані банківської картки.
5) Підтвердіть оплату квитків.
Expected Result: Система перевіряє дані, бронює квиток, відображає повідомлення про успішне бронювання і надсилає підтвердження користувачеві.


Сценарій 2: Реєстрація нового користувача в системі

Test Case: Успішна реєстрація нового користувача
Title: Успішна реєстрація нового користувача
Description: Користувач може успішно зареєструватися в системі, виконавши всі необхідні кроки.
Preconditions: Користувач знаходиться на сторінці реєстрації.
Test Steps:
1) Виберіть опцію "Зареєструватися".
2) Введіть особисті дані (ім'я, електронну пошту, пароль).
3) Натисніть кнопку "Зареєструватися".
4) Перейдіть по посиланню у листі для підтвердження реєстрації.
Expected Result: Система зберігає дані нового користувача, надсилає підтвердження реєстрації на електронну пошту і надає можливість користуватися всіма функціями системи.


Сценарій 3: Оформлення замовлення товару через онлайн-магазин

Test Case: Успішне оформлення замовлення
Title: Успішне оформлення замовлення
Description: Користувач може успішно оформити замовлення, виконавши всі необхідні кроки.
Preconditions: Користувач має товари у кошику і знаходиться на сторінці оформлення замовлення.
Test Steps:
1) Виберіть опцію "Оформити замовлення".
2) Введіть контактні дані для доставки.
3) Підтвердіть своє замовлення, натискаючи кнопку "Підтвердити замовлення".
Expected Result: Система зберігає дані замовлення, відображає повідомлення про успішне оформлення замовлення і надсилає підтвердження користувачеві.
